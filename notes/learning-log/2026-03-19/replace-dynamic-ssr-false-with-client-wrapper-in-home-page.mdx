## Context
`app/page.tsx` is a Server Component. Next.js 15 throws a build error when using `next/dynamic` with `ssr: false` inside it.

## Goal
Remove invalid dynamic import usage and keep terrain hero rendering on the homepage.

## Files Changed
- `app/page.tsx`

## Implementation Notes (key logic explained)
Removed `next/dynamic` and the `TerrainHero` dynamic declaration, then imported `HomeTerrainHero` (a Client Component wrapper) and rendered it at the top of the page.

## Key Decisions (and alternatives considered)
- Decision: keep `app/page.tsx` server-rendered and delegate browser-only behavior to a client wrapper.
- Alternative: convert `app/page.tsx` to client. Rejected due to larger rendering and performance impact.

## Assumptions
- Existing content in `app/page.tsx` should continue to be rendered as-is.

## Failure Modes
- If wrapper import path is incorrect, build fails with module-not-found.
- If `TerrainHero` runtime has browser-only issues, component may fail after hydration.

## How to Test (commands + steps)
1. Run `npm run dev`.
2. Confirm no build error about `ssr: false`.
3. Open `/` and confirm terrain hero appears and animates.
