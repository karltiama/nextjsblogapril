## Context
After fixing the Server Component dynamic import issue, production build still failed on a lint error in `terrain-hero.tsx`.

## Goal
Resolve the `react/no-unescaped-entities` lint error blocking the build.

## Files Changed
- `components/terrain-hero.tsx`

## Implementation Notes (key logic explained)
Escaped the apostrophe in the heading text from `I'm` to `I&apos;m` to satisfy JSX linting rules for unescaped entities.

## Key Decisions (and alternatives considered)
- Decision: use HTML entity escape in JSX text.
- Alternative: disable lint rule locally or globally. Rejected to keep lint quality and avoid masking similar issues.

## Assumptions
- Text content should remain exactly the same visually.

## Failure Modes
- If similar unescaped characters exist elsewhere, future builds may fail on additional lines.

## How to Test (commands + steps)
1. Run `npm run lint`.
2. Run `npm run build`.
3. Confirm no lint failure in `components/terrain-hero.tsx`.

## Next Steps
