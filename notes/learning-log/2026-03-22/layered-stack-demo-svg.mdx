## Implementation Notes (key logic explained)

- **Masking:** Each slice is an `absolute` wrapper with fixed `top`/`height` and `overflow-hidden`. The same `<img>` fills the card height (`WRAPPER_HEIGHT`) and uses `top: -sliceTop` so the strip that belongs at `sliceTop` in card space aligns with the clip window.
- **Motion:** `motion.div` wraps each slice; `y` comes from `getSliceY(index, activeLayer)` with collapsed vs two expanded tables.
- **Hover:** Three transparent divs match slice geometry; only indices 1 and 2 call `setActiveLayer`. Parent `onMouseLeave` clears to `null`. Top band intentionally has no `setState`.
- **Z-order:** Slice index 0 (top band) gets higher `z-index` so it reads as the top plate.

## Key Decisions (and alternatives considered)

- **`img` vs `next/image`:** Used `<img>` for a static public SVG and simple negative-offset layout; `next/image` would need explicit dimensions and offers little benefit for this SVG stack.
- **Spring transition:** `stiffness`/`damping` chosen for a quick, non-bouncy settle; can swap for `duration`-based tween in constants.

## Assumptions

- Asset URL `/viewportiso.svg` from `public/viewportiso.svg`.
- Slice windows and y-offset tables match the intended visual; tweak in constants only.

## Failure Modes

- Wrong `top`/`height` or negative image offset breaks band alignment.
- If hover zones do not cover slice rects, gaps could leave stale `activeLayer` until leave (zones match slice layout to avoid this).
