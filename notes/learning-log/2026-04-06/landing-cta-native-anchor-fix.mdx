## Context
The landing hero CTA buttons were updated to hash links, but user testing showed they still did not scroll reliably in the current sticky/3D hero layout.

## Goal
Make CTA click behavior deterministic so "Read the Blog" and "View Projects" always scroll to the correct sections on the landing page.

## Files Changed
- `app/page.tsx`

## Implementation Notes (key logic explained)
- Replaced `next/link` usage for the two hero CTAs with native `<a href="#...">` anchors:
  - `href="#blog-section"`
  - `href="#projects-section"`
- Kept existing styling classes so UI appearance remains unchanged.
- Left section IDs and `scroll-mt-24` in place from prior diff.

## Key Decisions (and alternatives considered)
- Chose native anchor behavior because it is browser-level and robust for same-page jumps.
- Alternative considered: client-side click handlers with `document.getElementById(...).scrollIntoView(...)`; rejected to avoid extra client JS and complexity.

## Assumptions
- The bug is specific to same-page hash behavior through App Router link handling in this page structure.

## Failure Modes
- If section IDs are renamed or removed, anchors will no longer target sections.
- If overlay layers intercept clicks, links can still fail regardless of anchor type.

## How to Test (commands + steps)
1. Run `npm run dev`
2. Open `/`
3. Click `Read the Blog` and confirm jump to `#blog-section`
4. Click `View Projects` and confirm jump to `#projects-section`
5. Repeat in mobile and desktop viewport sizes

## Next Steps
- If click issues persist, next minimal diff is to set `pointer-events-none` on the Canvas wrapper and keep `pointer-events-auto` only on CTA content.
- Optionally add smooth scroll (`scroll-behavior: smooth`) in global styles for improved UX.
