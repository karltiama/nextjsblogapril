## Context

Site accent links use Tailwind `text-blue-400` / `hover:text-blue-300` (e.g. `PostItem`, home “View all posts”). The main nav used `hover:text-primary`, but `--primary` in `globals.css` is a green/teal brand color, so header hovers did not match the blue used elsewhere.

## Goal

Make header interactive elements on hover use the same blue treatment as the rest of the site.

## Files Changed

- `components/main-nav.tsx` — logo + Blog/Projects/About link hovers
- `components/site-header.tsx` — GitHub, Twitter, Strava ghost icon hovers
- `components/mode-toggle.tsx` — theme toggle ghost button hovers
- `components/mobile-nav.tsx` — mobile menu trigger outline hover

## Implementation Notes (key logic explained)

- Replaced `hover:text-primary` with `hover:text-blue-400` on desktop nav links and added `transition-colors hover:text-blue-400` on the home/logo link.
- Ghost icon buttons kept `buttonVariants({ variant: "ghost" })` and appended `hover:bg-blue-400/10 hover:text-blue-400` to mirror the home page ghost CTA pattern (`hover:bg-blue-400/10`).
- Mobile menu uses `outline` variant; added `hover:border-blue-400/50 hover:text-blue-400` so the icon picks up blue without changing the global button component.

## Key Decisions (and alternatives considered)

- **Tailwind blues vs CSS variable:** Could have introduced `--link-accent` in `globals.css` for one source of truth; skipped to stay consistent with existing `blue-400` usage across components.
- **Not changing global `ghost` variant:** Avoids unintended styling changes on every ghost button in the app.

## Assumptions

- “Same blue as everything else” means the existing `blue-400` / light hover background pattern, not the theme `primary` color.

## Failure Modes

- If Tailwind merge order ever drops the hover overrides, ghost buttons could revert to accent gray; re-check `cn()` / tailwind-merge if that appears.
- Outline button default hover may still apply alongside the new classes; visually verify the menu icon on mobile widths.

## How to Test (commands + steps)

1. `npm run dev` (or your usual dev command).
2. Desktop: hover Blog, Projects, About, site title/logo, social icons, theme toggle — text/icon should shift to blue with a subtle blue-tinted background on icon buttons.
3. Narrow viewport: open mobile menu trigger; border and icon should hint blue on hover.
