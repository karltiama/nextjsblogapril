## Context
Hero CTA anchor links still did not work after switching to native anchors, indicating click events might be blocked by an overlaying layer in the 3D hero.

## Goal
Ensure hero buttons reliably receive click events by preventing the canvas layer from intercepting pointer input.

## Files Changed
- `components/terrain-hero.tsx`

## Implementation Notes (key logic explained)
- Added `pointer-events-none` to the absolute canvas wrapper:
  - from `className="absolute inset-0"`
  - to `className="absolute inset-0 pointer-events-none"`
- This keeps visual rendering intact while allowing pointer interaction to pass through to interactive content above.

## Key Decisions (and alternatives considered)
- Chose CSS-level event passthrough as the smallest safe fix.
- Alternative considered: increasing z-index on CTA wrapper only; rejected because it can still be brittle with layered/sticky contexts.

## Assumptions
- Canvas overlay was intercepting clicks in the hero region.

## Failure Modes
- If another overlay layer still has pointer events enabled above CTAs, clicks can remain blocked.
- If click works but scroll still fails, issue is then anchor target or scroll behavior, not pointer interception.

## How to Test (commands + steps)
1. Run `npm run dev`
2. Hard refresh `/` (Ctrl+Shift+R)
3. Click `Read the Blog` and confirm jump to `#blog-section`
4. Click `View Projects` and confirm jump to `#projects-section`
5. Verify clicks work on mobile and desktop viewport widths

## Next Steps
- If still broken, add explicit client-side `scrollIntoView` handlers in a dedicated CTA client component.
- Add an e2e regression test for CTA click + scroll behavior.
