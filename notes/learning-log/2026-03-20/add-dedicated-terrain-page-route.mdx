## Context
Terrain hero should be accessible on its own page instead of the home route.

## Goal
Create a dedicated route that displays the terrain hero full-screen.

## Files Changed
- `app/terrain/page.tsx`

## Implementation Notes (key logic explained)
Added a new App Router page at `/terrain` that renders `HomeTerrainHero`. Included a simple overlay `Back home` link in the top-left so navigation remains easy.

## Key Decisions (and alternatives considered)
- Decision: reuse existing `HomeTerrainHero` wrapper for client-safe loading.
- Alternative: import `TerrainHero` directly in this page. Not chosen to avoid duplicating client-loading logic.

## Assumptions
- Route path `/terrain` is acceptable and not currently used.

## Failure Modes
- If route path conflicts with future content, page organization may need renaming.

## How to Test (commands + steps)
1. Run `npm run dev`.
2. Open `/terrain`.
3. Confirm wireframe terrain displays and animates.
4. Click `Back home` and confirm navigation to `/`.

## Next Steps
