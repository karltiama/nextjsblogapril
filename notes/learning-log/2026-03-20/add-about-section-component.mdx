## Context

The user provided an `AboutSection` design: dark grid background, staggered scroll-in copy, stats row, and integration with `AboutHeadline`.

## Goal

Ship `AboutSection` as a client component using `useInView` and the shared `stagger` helper pattern.

## Files Changed

- `components/about-section.tsx`

## Implementation Notes (key logic explained)

- `useInView(0.1)` drives `inView`; each block uses `stagger(i, className)` to append `animate-[fade-in-up_0.6s_ease-out_both]` and `animationDelay` based on index × `STAGGER_MS` (80ms).
- Apostrophes escaped for ESLint (`I&apos;m`, `it&apos;s`).
- Section `id="about"` for deep links / nav.

## Key Decisions (and alternatives considered)

- **Decision:** keep `AboutHeadline` as separate component per user sketch.
- **Alternative:** inline headline in `AboutSection` — rejected to mirror provided structure.

## Assumptions

- Dark `#0a0a0f` section is intentional even if the rest of the about page uses theme colors.

## Failure Modes

- Below-the-fold first paint shows `opacity-0` until intersection — expected.

## How to Test (commands + steps)

1. `npm run dev` → `/about`
2. Scroll from top: label → headline → paragraphs → stats should stagger in.
