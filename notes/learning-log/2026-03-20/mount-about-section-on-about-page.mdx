## Context

The About route already had a long portfolio-style layout; the new hero-style `AboutSection` should appear first.

## Goal

Render `AboutSection` above the existing `container` content on `/about`.

## Files Changed

- `app/about/page.tsx`

## Implementation Notes (key logic explained)

- Imported `AboutSection` and wrapped the page return in a fragment.
- Placed `<AboutSection />` before the existing `div.container` so the dark full-bleed section sits on top; legacy content remains below.

## Key Decisions (and alternatives considered)

- **Decision:** keep existing about body instead of replacing the whole page — preserves metrics, skills, and CTAs.
- **Alternative:** replace page with only `AboutSection` — rejected unless user asks to drop the rest.

## Assumptions

- Two “about” experiences (hero + detailed) on one page is acceptable.

## Failure Modes

- Possible duplicate “About” narrative; user can trim lower sections later.

## How to Test (commands + steps)

1. `npm run dev` → `/about`
2. Confirm dark section at top, then existing cards/content below.
