## Context
Browser console showed: `THREE.Clock` deprecation warning while rendering terrain with React Three Fiber.

## Goal
Reduce deprecation noise by pinning `three` to an earlier compatible release line with current Fiber setup.

## Files Changed
- `package.json`
- `package-lock.json`

## Implementation Notes (key logic explained)
- Pinned:
  - `three` -> `0.179.1`
  - `@types/three` -> `0.179.0`
- Kept `@react-three/fiber` at `9.5.0`.
- Verified dependency tree resolves to the pinned versions and build still succeeds.

## Key Decisions (and alternatives considered)
- Decision: downgrade `three` minor line first (smallest runtime change) instead of rewriting terrain internals.
- Alternative: refactor app code to use `THREE.Timer` directly. Not applicable because warning is from library internals, not our component code.

## Assumptions
- The deprecation originates from Three/Fiber interaction and is sensitive to `three` version.

## Failure Modes
- Warning may still appear if Fiber internals call `Clock` regardless of Three minor version.
- Future dependency upgrades can reintroduce warning.

## How to Test (commands + steps)
1. `npm run dev`
2. Open `/terrain`
3. Check browser console for `THREE.Clock` warning.
4. Run `npm run build` (already passing).

## Next Steps
- If warning persists, pin Fiber to a version explicitly known to use `Timer`, or accept warning until upstream patch.
