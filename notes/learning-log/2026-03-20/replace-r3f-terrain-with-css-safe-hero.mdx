## Context
Homepage kept crashing at runtime with `Cannot read properties of undefined (reading 'ReactCurrentOwner')` coming from `@react-three/fiber`.

## Goal
Keep a terrain-style hero on the home page while removing the unstable React Three Fiber runtime path.

## Files Changed
- `components/terrain-hero.tsx`

## Implementation Notes (key logic explained)
Replaced the 3D Fiber/Three implementation with a lightweight client-safe hero:
- layered gradient background
- perspective-styled animated horizontal lines
- same `TerrainHero` API (`children`, `fullHeight`) so `app/page.tsx` integration remains unchanged

This preserves the visual “terrain” feel without importing Fiber internals.

## Key Decisions (and alternatives considered)
- Decision: replace runtime implementation with CSS-based terrain effect for stability.
- Alternative: continue debugging package/bundler internals for Fiber in this stack. Rejected for now due to repeated runtime failures.

## Assumptions
- Visual terrain effect is acceptable even if it is not true WebGL geometry.

## Failure Modes
- Visual style may differ from original 3D wireframe expectations.
- Animation performance can vary on low-end devices if too many lines are rendered.

## How to Test (commands + steps)
1. Run `npm run dev:clean`.
2. Open `/`.
3. Confirm no `ReactCurrentOwner` runtime error.
4. Confirm hero animation and text overlay render correctly.

## Next Steps
- If you still want real WebGL terrain later, reintroduce it in an isolated route or a dedicated client-only sandbox with pinned-compatible versions.
