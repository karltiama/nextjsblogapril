## Context
`/terrain` still threw `ReactCurrentOwner` runtime errors even with client-only routing. Dependency audit showed nested invalid `react@19/react-dom@19` from `@mermaid-js/mermaid-cli -> @zenuml/core`.

## Goal
Eliminate React version conflict sources that can break React Three Fiber runtime internals.

## Files Changed
- `package.json`
- `package-lock.json`

## Implementation Notes (key logic explained)
Removed `@mermaid-js/mermaid-cli` from project dependencies. This package is a CLI/tooling dependency and introduced a React 19 subtree conflicting with app React 18 runtime assumptions.

## Key Decisions (and alternatives considered)
- Decision: remove `@mermaid-js/mermaid-cli` from installed app deps.
- Alternative: keep it with overrides. Not reliable here because invalid nested versions persisted and still surfaced in runtime instability.

## Assumptions
- Mermaid CLI is not required at app runtime for page rendering.

## Failure Modes
- If you relied on local `mmdc` CLI commands from this repo, those commands will no longer be available until reinstalled separately.

## How to Test (commands + steps)
1. Run `npm run dev:clean`.
2. Open `/terrain`.
3. Confirm no `ReactCurrentOwner` error.
4. Confirm wireframe terrain animates.

## Next Steps
- If Mermaid CLI is still needed, install it globally or in a separate tooling workspace to isolate React versions.
