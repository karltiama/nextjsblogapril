## Context

The About section was not visibly peeking enough in the first viewport on the terrain page.

## Goal

Make the `#about` section clearly visible within the initial viewport while preserving the hero effect.

## Files Changed

- `components/terrain-hero.tsx`

## Implementation Notes (key logic explained)

- Increased bridge offset effect by reducing top padding in full-height mode:
  - `-mt-[100svh] pt-[92svh]` -> `-mt-[100svh] pt-[84svh]`
- Reduced transition spacer height:
  - `h-28` -> `h-10`
- Net effect: content below starts earlier and the About section peeks more aggressively above the fold.

## Key Decisions (and alternatives considered)

- Chosen: `pt-[84svh]` for a clear, noticeable peek.
- Alternatives:
  - `pt-[88svh]` for moderate peek
  - `pt-[90svh]` for subtle peek

## Assumptions

- Terrain route should prioritize showing a continuation cue over a fully isolated hero.

## Failure Modes

- Too much reveal may reduce hero immersion on smaller screens.
- Very short viewport heights could feel crowded at the fold.

## How to Test (commands + steps)

1. Run `npm run dev`.
2. Open `/terrain`.
3. Verify `#about` text is visible in first viewport without scrolling.
4. Resize browser height and confirm the fold remains visually balanced.

## Next Steps

- If too aggressive, increase `pt` to `86svh` or `88svh`.
- If still too subtle, reduce `pt` to `82svh`.

## Verification Checklist

- [ ] About section is visible in initial viewport.
- [ ] Hero still reads as primary visual.
- [ ] Transition between hero and About remains smooth.
